Why Hadoop?

HDFS & YARN

Hadoop 的核心精神 --
"如果一台機器無法處理，那就用多台一起下去處理"

HDFS

  • 將大資料切分成好幾小塊
  • 每一小塊複製 n 份後分給不同機器儲存
    若部分資料遭損毀，仍然能在其他機器找回複製過的資料
  • 展現 HDFS 高容錯率與平衡負載機制

YARN

  • 管理運算資源，記憶體 RAM 以及核心 CPU
  • 在不同電腦上執行程式
    Mapreduce 資料處理引擎會將結果彙整

Spark on Yarn

與 Mapreduce 同為資料處理引擎，只不過處理效能更加快速有效率

與 Hadoop 相容

Spark 可以結合 Hadoop 叢集，並透過 Yarn 來管理我們的運算資源以及存取 HDFS 分散式檔案系統內的資料

比 Mapreduce 更加快速

Spark 是基於記憶體運算，而 Mapreduce 是將讀取的資料先暫存到硬碟中(SSD)，很容易就會有讀寫資料延遲的問題
再加上 Spark 底層有做演算法最佳化，可以在執行程式時自動讓本身效能達到最好的效果

PySpark API

Spark 支援多種語言，包括資料科學界的主力語言 python，大大提升資料科學家們在使用上的便利性
